ℹ️ Logique de sécurité SSL VPN

Pour que SSL VPN soit considéré comme sécurisé, deux options sont disponibles :

Option 1 : Désactiver SSL VPN sur les zones WAN (recommandé si non utilisé)

Option 2 : Si SSL VPN sur WAN est nécessaire, les conditions suivantes doivent être remplies :
• Local Users doit être résolu (mots de passe changés, TOTP retiré)
• LDAP doit être résolu (si configuré) - bind password changé
• RADIUS doit être résolu (si configuré) - secrets changés
• TACACS+ doit être résolu (si configuré) - secrets changés

ℹ️ Note importante sur la remediation CSE

Suite à la compromission des backups SonicWall, vous devez dissocier manuellement le Firewall Connector dans MySonicWall avant de pouvoir réactiver CSE. Une fois l'action manuelle effectuée, vous devrez attendre 30 secondes avant de pouvoir réactiver CSE automatiquement via l'interface.

📋 Étapes manuelles dans MySonicWall :

  1. Connectez-vous à MySonicWall
  2. Allez dans Products et sélectionnez le Tenant
  3. Recherchez le produit "SonicWall Cloud Secure Edge"
  4. Dans les détails, cliquez sur "Firewall Connection"
  5. Cliquez sur l'icône poubelle pour dissocier le connector
